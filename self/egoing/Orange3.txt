# 6-1 통계와 시각화
통계의 핵심: 데이터를 이해하는 것
이를 위해 인류는 무한히 노력
그 노력을 한마디로? "어림짐작"
어림짐작을 위한 여러 도구들이 있는데
그 중 가장 위대한 도구 : 대푯값, 분포

데이터를 한 눈에 파악할 수 있느 간단한 단 하나의 숫자를 원했다
데이터에 대한 첫인상 같은 것
가장 유명한 것 : 평균
가장 가운데 있는 값을 의미하는 중앙값,
가장 자주 등장하는 값을 의미하는 최빈값 같은 것들이 있다
데이터들을 대표하기 때문에 대푯값이라고도 한다

평균 : 값들의 합 / 값의 개수
중앙값 : 값들 중 중앙에 위치한 값

평균과 중앙값은 항상 같은가? 아니다

대푯값만으로는 데이터 성격을 완전 파악하긴 힘들다
데이터가 왼쪽으로 좁게 오른쪽으로 넓게 분포할 수도 있다
따라서 값들의 분포를 확인하기 위해서
사분위수 quartile, 표준편차 등의 개념이 필요하다

오렌지를 이용하면 이런 통계적인 작업을 시각적인 도구와 함께 확인할 수 있다
그 도구는 "box plot" 이라고 한다

# 7-1 머신러닝
온도와 판매량의 관계를 수학적으로 나타낸다면?
y = 2x : 온도에 따른 판매량의 변화 공식
공식이 있다면 그래프를 보지 않아도 예측이 가능

원인이 온도뿐만이 아니라면?
계절, 경쟁자의 수, 요일 등등의 데이터들도 판매량에 영향을 미칠 수 있다
그러면 온도 말고도 다른 열의 데이터도 x축에 표현해야 할 것이다
하지만 평면에선 축이 부족하다
3차원, 4차원, .... 현실세계에서 그림으로 표현할 수 없게 됨
또 원인과 결과의 그래프가 위의 예처럼 직선처럼 단순하지 않고 엄청 복잡한 게 현실인데
그 안에서의 규칙성을 찾고 공식을 만들어내기란 인간의 힘만으론 힘들고 불가능 할 수도

원인과 결과를 컴퓨터 스스로 파악할 수 있다면?
그런 꿈을 가진 사람들에 의해 만들어진 기술 : 머신러닝

# 7-2 머신러닝 이용하기
file 더블클릭 - 컬럼 정보
role : feature, target, meta, skip

skip :  분석작업에서 무시해도 되는 데이터
meta : 실제 분석작업에 사용되진 않지만 정보성으로 남겨두면 좋은 데이터
target : 예측하고자하는 데이터(종속변수, 결과)
feature : 예측하는데 사용할 데이터(독립변수, 원인)

특징들을 학습시켜서 원하는 타겟을 구하는 것이 목표

model : 여러 모델 사용 가능
evaluate : test, prediction 등

# 8 수업을 마치며


오렌지의 가장 큰 장점은 GUI 기반, 동시에 단점도 GUI 기반이라는 것
GUI는 마우스나 손가락 터치 등으로 사물을 다루듯이 컴퓨터를 조작하는 방법을 의미
GUI의 경쟁자는 코딩 
둘 다 같은 일을 하지만 장점, 단점이 다르다
오렌지는 내가 할 수 있는 일이 아이콘의 형태로 존재하기 때문에 모든 것을 기억할 필요는 없다
이미 알고 있는 것이라면 아이콘을 보는 것만으로도 내가 이걸 사용하는데 필요한 정보들이 연상되기 때문이다
모르는 것도 눈에 보이기 때문에 앞으로 알아야 할 것임을 인식하고 학습방향을 잡아갈 수 있다
이런 장점을 최대한 활용해 학습에 적용해라 그런 점에서 오렌지는 정말 좋다

반면 코딩은 언어적이다. 문법을 배워야 함, 어휘를 기억해야 함
따라서 배우는 노력이 많이 필요하고 하지만 검색을 통해 필요한 기능을 찾아내기 매우 쉽다
또 GUI보다 훨씬 복잡하고 논리적인 흐름을 만들 수 있다
오렌지의 아이콘이 1만개라면?
반면 1만줄짜리 코드는 코딩의 세계에서 아무 것도 아니다
같은 일을 다른 방법으로 하는 두 개의 기술이 모두 존재한다는 것은
모두 각자의 쓰임이 있다는 것
사용자인 우리가 해야 할 일은
두 개의 장점만을 취사선택해서 장점만 가진 환상을 만들어내는 것
매우 복잡한 작업을 처리해야 한다, 보다 자동화된 처리를 해야한다면 코딩, 프로그래밍이라는 것을 배워라


자꾸 사용하지 않고 공부만 하는 이유
무언가를 사용하는 것이 사실 쉬운 일이 아니기 때문
그래서 자꾸 공부만 한다
공부할수록 사용하는 것은 점점 더 어려워진다
너무 공부만 하지 말고 즐기면서 삶에 녹여보길
