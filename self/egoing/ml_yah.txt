# 1-3 꿈
우리는 누군가에게 평가받기 위함이 아니라
나의 문제를 머신러닝으로 해결해보기 위해 모였다

학생이 아니라 엔지니어
소비자가 아니라 생산자가 되기 위해 만난 것이다

우리에게 제일 절실한 것 : 해결하고자하는 문제!
해결해고자 하는 문제가 없다면 지식은 목적 없는 수단에 불과

각자 중요하게 생각하는 문제가 모두 다를 것
해결하고자 할 문제가 없을지도 모르고

수업에서 문제를 빌려주겠다
지구상에 살았고 살고 있고 살게 될 모든 사람이 빠짐없이 경험하고 있는 오래된 문제
바로 고쳐지지 않는 '습관'
습관을 고치는 데에 머신러닝을 이용해보면 어떨까

내가 해야 할 일은 단지 가상의 문제를 크고 심각하고 절망적으로 과장하는 것
어려울 것이 하나도 없다
나는 어떤 습관을 가지고 있는가
ex 손톱 깨물기

습관을 너무 바꾸고 싶다
번번히 실패
어려운 싸움에는 지략이 필요
전략을 세워보자

우선 습관이라는 것에 대해 생각해보자
아무리 발버둥을 쳐도 내 의지만으로는 습관을 이기기가 쉽지가 않다
제아무리 깊게 뿌리박힌 습관이라도 그 습관에 지속적으로 영향을 주는 환경이 바뀌면
습관이서서히 바뀐다
환경을 바꾸는 건 나의 의지이다

- 습관은 의지를 이긴다
- 의지는 환경을 이긴다
- 환경은 습관을 이긴다

의지만으로 습관을 바꾸는 건 무모한 일인 듯
의지로 환경을 조정해서 환경이 습관을 손 봐주는 우회전략을 구사하면 좋지 않을까

내가 손톱을 깨물면 손톱을 깨물고 있다는 것을 알려줄 수 있느 ㄴ새로운 환경을 만들고 싶다는 의지가 생김
의지가 아무리 강해도 환경을 바꾸는 건 쉽지가 않다
누군가가 손톱을 계속 깨물고 있는지 나를 계속 지켜 보고 있어야 한다
또 소놉을 꺠물면 그것을 알려주는 환경을 만들어야 하는데 쉽지 않다
이 환경을 만드는데 머신러닝을 이용할 수 없을까?
손톱을 깨물면 그것을 나한테 알려주는 환경을 만들고 싶어졌다!
이런 환경에 지속적으로 노출되면 서서히 습관을 새로고침 할 수 있을 것 같다!

# 1-4 궁리한느 습관
일은 두 가지 요소로 이뤄져 있다
 == (하고 싶은 것)꿈 + (꿈을 이뤄낼 수 있는)능력
꿈은 능력이 있어야 현실이 될 수 있고
능력은 꿈이 없다면 현실에 존재할 필요가 없다
꿈과 능력은 육체와 영혼처럼 따로 분리할 수가 없다

우린 아직 손톱을 깨물면 알려줄 수 있는 것을 만들 능력은 없다, 엔지니어가 아니므로
만들고 싶다는 꿈만 있는 상태

이럴수록 능력이 있다고 간주를 한 상태에서 꿈을 먼저 꾸는 것이 중요하다
우리의 꿈이 능력에 주눅들지 않도록 하기 위해 수업에서 3가지를 철저히 감추는 전략을 세움
1. 원리
2. 수학
3. 코딩

믈론, 머신러닝은 원리를 이용해서 수학과 코딩으로 이루어진 것이다
머신러닝으로 궁극의 목적을 달성하기 위해서는 수학과 코딩이 언젠가는 필요해질 것이다
하지만 처음부터 필요한 것은 아니다

스마트폰을 사용하기 위해서 스마트폰 제조 방법을 꼭 알아야 할까

더 좋은 스마트폰을 만들고 싶다면 그 때 제조방법을 배워도 될 것이다

머신러닝의 사용자가 우선 되어보자
사용자가 되어서 머신러닝으로 다양한 문제를 해결하다 보면 한계를 느끼게 될 것이다
그 한계가 충분히 절망스러울 때 수학과 코딩을 공부해도 늦지 않다

꿈만 꾸는 사람은 종종 몽상가로 치부되곤 한다
하지만 과학기술이 발전되면서 꿈을 실현하는 방법이  빨라지고 저렴해지고 간단해지고있다
꿈이 있는 사람이 몽상가가 아닌 혁명가가 될 수 있는 시대가 오고 있는 것


꿈을 꾸자, 계획을 세워 보자

잠에서 깨어나 보니까 이런저런 상념이 떠오를 떄가 있다

# 1-5 Teachable Machine
머신러닝을 위해서 꼭 수학과 코딩을 알아야 하는 것은 아니다
두 가지 없이도 머신러닝을 이용할 수 있게 해 주는 서비스들이 생겨나고 있다

미래에는 수학자나 프로그래머가 아니라도 누구나 머신러닝을 이용해서 기계를학습시키고
그것을 활용해서 자신의 문제를 해결할 수 있을 것

마치 컴퓨터가 소수의 과학자들만 다루는 음흉한 비밀무기처럼 인식되었지만
오늘날은 모든 사람들이 전화기의 탈을 쓴 스마트폰과 씨름하면서 컴퓨팅을 하고 있는 것처럼
머신러닝도 그렇게 될 것이다

그 가능성의 사례 : Teachable Machine
머신러닝에 대한 지식 없이도 머신러닝을 이용할 수 있도록 고안된 도구

Class : 분류
손톱 깨무는 사진 / 아닌 사진을 학습시키기 위해 모았음
즉, 컴퓨터가 학습할 교재를 만든 것!
데이터를 충분히 입력했다면 학습을 시켜본다

기계가 학습을 제대로 했는지 평가하는 것이 중요한데
얼굴 각도를 바꿔 보면 손톱 정상 헷갈려 함
학습 자료가 부족하거나 부정확하기 때문
더 많은 사진을 입력하거나 부정확한 장면을 제거하면 된다

사람이 경험과 공부를 통해 더욱 정교한 판단력을 갖게 되는 것과 비슷하다
사람이 학습하듯 기계를 학습시켜 정확한 판단력을 갖게한다는 점에서
이러한 기술을 기계학습, machine learning 이라 부른다

신기할 것
신기해 하는 것 == 생산자 되는 데에 있어 가장 중요한 재능
신기하지 않다면
신기해 하기 위해 노력해라 -> 후천적으로도 충분히 키울 수 있는 능력!!!

손톱 꺠무는지를 구분할 수 있는 판단력을 기계로 구현
이 판단력은 놀랍게도 파일로 만들어 다운로드 할 수 있다
export model -> download

독립변수는 원인이 되는 열
종속변수는 원인으로 인해 결과가 되는 열

온도가 20도일 때 40잔이 팔렸다
원인은 온도 20도
결과는 판매량 40잔

원인은 결과와 상관 없이 일어나는 사건이다
판매량 때문에 온도가 달라질 리가 없다
결과에 영향을 받지 않는 독립적인 사건

그래서 원인은 독립변수, 결과는 종속변수

특성 == 열

따라서 온도와 판매량은 상관관계이다

온도-판매량 사이 미묘한 관계가 있다
- 온도와 판매량이 같이 커지고, 작아지고 있음을 발견함
- 온도와 판매량 사이에 다른 영향을 줄 만한 다른 특성이 발견되지 않는다
- 판매량이 달라지니 온도가 달라졌다? - 이상, 온도가 달라지니 판매량이 달라졌다? - 가능
- 온도의 2배가 판매량이 되는 일정한 패턴이 발견!

결론적으로, 온도는 원인, 판매량은 결과라 할 수 있다
각 열이 원인과 결과의 관계일 때 그 두 개의 열은 인과관계에 있다고 한다

상관관계와 인과관계는 비슷해 보이지만 중요한 차이가 있다

상관관계는 인과관계를 포함한다
모든 인과관계는 상관관계이다
모든 상관관계가 인과관계인 것은 아니다
특성들 사이의 관계를 파악하는 것은 매우 어렵고 조심스럽게 접근해야 한다
적은 수의 데이터를 가지고 상관관계가 있다고 단정하면 안 된다
그리고 단지 상관관계가 있을 뿐인데 그것을 인과관계로 단정 지으면 안 된다


특성들 사이의 관계를 파악하는 일이란, 실전에서 매우 엄격한 기준을 가지고
다양한 통계기법을 동원해야하는 고되고 중요한 작업!

기계학습은 지도학습, 비지도학습, 강화학습으로 이뤄져 있다

지도학습은 분류, 회귀

지도학습 supervised learning
지도 : 기계를 가르친다
문제집을 푸는 것과 비슷
문제집엔 문제가 있고 정답이 있다
문제와 정답을 비교하고 맞히다 보면 점점 문제 푸는 것에 익숙해진다
이후에는 비슷한 문제를 마주치면 오답에 빠질 확률이 점점 낮아진다
문제집으로 학생을 가르치듯 데이터로 컴퓨터를 학습시켜서 모델을 만드는 방식을 지도학습이라고 한다

앞서 살펴본 손톱 감시 앱, 레몬 판매량 예측 작업은
지도학습을 이용한 것이다

비지도학습 unsupervised learning은 지도학습에 포함되지 않는 방법들이다
여기 속하는 도구들은 대체로 기계에게 데이터에 대한 통찰력을 부여하는 것이라 이야기할 수 있다
통찰의 사전적 의미 : 예리한 관찰력으로 사물을 꿰뚫어 봄
누가 정답을 알려주지 않았는데도 무언가를 관찰하는 것을 통해 새로운 의미, 관계를 밝혀내는 것이라 할 수 있다
데이터 성격을 파악하거나 데이터를 잘 정리정돈 하는 것에 주로 사용된다

강화학습 reinforcement learning
강화학습은 학습을 통해서 능력을 향상시킨다는 점에서는 지도학습과 비슷
차이점은 지도학습은 정답을 알려주는 문제집이 있는 것이라면
강화학습은 어떻게 하는 것이 더 좋은 결과를 낼 수 있는지 스스로 느끼면서
실력 향상을 위해 노력하는 수련과 비슷
경험을 통해 더 좋은 답을 찾아가는 것
마치 게임 실력을 키워가는 것처럼
게임에는 규칙이 있다, 그 규칙에 따라 어떤 행동을 하면 그 결과에 따라 상이나 벌을 받는다
더 큰 상을 받기 위한 과정을 끊임없이 반복하다보면 그 게임의 고수가 된다
이 과정을 기계에게 시켜서 기계 스스로가 고수가 되도록 고안된 방법

# 1-16 지도학습
지도학습은 역사와 비슷하다
역사는 과거에 있었던 일이 원인과 결과로 기록되어 있다
역사를 알면 어떠한 사건이 일어났을 때 그것의 결과로 어떤 일이 일어날 지 예측할 수 있게 된다
마찬가지로 지도학습은 과거의 데이터로부터 학습해서 결과를 예측하는 데 주로 사용된다

궁금한 것은 1월 8일은 몇 개가 판매 될지이다

이 때 머신러닝의 지도학습이 이용될 수 있다

이용하기 위해선 우선 충분한 데이터가 수집되어야 한다
또 데이터는 원인이 독립변수와 결과인 종속변수로 이뤄져 있어야 한다
이것을 지도학습으로 훈련시키면 컴퓨터는 모델을 만든다
아마 모델은 이렇게 생겼을 것 
온도 X 2 = 판매량

일단 모델이 만들어지면 모델을 사용하면 된다

8일에는 25 X 2 = 50 개가 판매될 것을 예측

머신러닝의 지도학습을 이용하면 온도 x2 라는 모델을 컴퓨터가 알아서 만들어준다

# 1-21 비지도학습 정리
비지도학습은 탐험적
탐험이 미지의 세계를 파악하는 것이듯이
데이터의 성격을 파악하는 것이 목적이다
독립변수와 종속변수의 구분이 중요하지 않다
그냥 데이터만 있으면 된다

지도학습은 역사적이다
과거의 원인과 결과를 바탕으로 결과를 모르는 원인이 발생했을 때 그것이 어떤 결과를 초래할 것인지를
추측하는 것이 목적이다
그래서 원인인 독립변수와 종속변수가 꼭 필요하다

잠깐 엉뚱한 이야기...
나는 누구인가를 파악하는 좋은 방법이 있다
내가 하는 말들의 빈도수를 정리정돈해서 조사해보는 것
긍정적 단어를 많이 사용했다면 나는 긍정적인 사람일 확률이 높다
집안의 물건들을 정리정돈해보는 것도 나를 파악하는 좋은 방법 중 하나

정리정돈해보면 대상의 성질을 파악할 수 있다
비지도학습을 한마디로 정리하는 것은 어렵지만
군집화와 연관규칙만을 두고 봤을 때
데이터의 성격을 파악하는 작업은 결국 비슷한 것은 모으고 다른 것은 떨어뜨리는 것이다
즉 그룹핑을 하는 것이다
비지도학습은 데이터를 정리정돈해서 그 표에 담긴 데이터의 성격을 파악하는 것이 중요한 목적

변환은? 까치밥


# 1-22 강화학습
강화학습 reinforcement learning 
강화학습의 핵심은 일단 해 보는 것

지도학습은 배움을 통해서 실력을 키우는 것이라면
강화학습은 경험을 통해서 실력을 키워나가는 것이다

행동의 결과가 자신에게 유리한 것이라면 상을 받고 불리한 것이라면 벌을 받는다
이 과정을 매우 많이 반복하면 더 많은 보상을 받을 수 있는 더 좋은 답을 찾아낼 수 있다는 것이
강화학습의 기본 아이디어

비유!
강화학습은 게임의 실력을 키우는 것과 비슷하다

두 개의 주체를 생각해보자
게임과 게이머
게임 은 게이머에게 보여줄 화면이 필요
게이머 는 우선 현재의 상태를 관찰해야 한다
관찰의 결과에 따라서 게임을 조작하는 행동을 해야 하고
관찰과 행동을 하기 위해서는 판단력이 필요하다


게임은 게이머에게 현재 상태를 보여준다
캐릭터는 어디에 있고 장애물은 어디에 있는지 보여준다
동시에 현재의 점수도 알려준다
게이머는 이 점수가 높아지는 것이 상이고
자앵물에 부딪히는 것이 벌이다
관찰의 결과에 따라서 어떤 상태에서 어떻게 행동해야 더 많은 상을 받고 더 적은 벌을 받을 수 있는지 알게 된다
즉 판단력이 강화된다

판단에 따라서 행동을 하게 되고 그 행동은 게임에 변화를 주게 된다
이 과정을 반복하면 판단력이 강화된다


배우지 않고 잘하게 되는 많은 일들이 이러한 과정을 통해서 이루어진다
강화학습은 이러한 과정을 모방해서 기계를 학습시키는 것이다

강화학습의 용어로 바꿔보자
게임 == 환경 environment
게이머 == 에이전트 agent
상태 == 상태 state
상 == 보상 reward
판단 == 정책 policy
행동 == 행동 action

즉 강화학습은 상태에 따라서 더 많은 보상을 받을 수 있는 행동을 
에이전트가 할 수 있도록 하는 정책을 만드는 것이 목적이다
