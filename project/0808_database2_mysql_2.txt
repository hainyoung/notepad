# 20. MySQL Workbench

workbench 설치를 위해서 visual studio 2019를 깔고
c++ 개발 환경으로 설치한다

mysql monitor를 쓰려면
명령어로 mysql을 적고
-uroot -p라 적어줘야하는데
-hloalhost도 적는데, 
h는 hostd의 약자로,
host는 인터넷에 연결되어 있는 각각의 컴퓨터를 말한다

그래서
우리가 실행하려고하는 mysql monitor라는 client로
인터넷을 통해서 다른 컴퓨터에 있는 mysql server에 접속하려고 하면
-hlocalhost 대신 -hgoogle.com 이라고 적어줘야 한다(구글 직원이라면)

현재 우리는 mysql client와 mysql server가 같은 컴퓨터에 위치하고 있음
지금 우리가 사용하고 있는 mysql client가 설치되어 있는
자신의 컴퓨터를 가리키는 약속된 특수한 도메인이 있는데 그것이 localhost이다
같은 의미의 ip 주소는 127.0.0.1이다

-hlocalhost == -h127.0.0.1
그런데 우리가 -hloclhost 부분을 입력하지 않아도 디폴트로 반영이 된다
암묵적으로 mysql monitor와 mysql monitor가 설치되어 있는 컴퓨터의 mysql server를 가리킴


workbench를 사용해보자

apply -> sql문이 나온다
뭔 뜻?
mysql monitor를 쓰건 mysql workbench를 쓰건
모든 client들은
결국에는 sql을 mysql server에 전송함으로써
데이터베이스 서버를 제어하게 된다!
-> workbench(mysql slient)는 sql문을 만들어서 서버에 전달해준다
이 사실을 이해하는 것이 나중에 python, php, java 컴퓨터 프로그래밍 언어를 통해
app, web을 만들 때 굉장히 중요할 것! 개념적 이해 필수!

컴퓨터 프로그래밍 언어를 이용해서 우리가 하는 일은, 
sql문을 생성해서 그것을 server에게 전달하는 것이다

mysql 서버를 중심으로 해서
다양한 mysql client생태계가 조성되어 있다
세상에는 정말 많은 mysql client들이있다
mysql server를 사용하고 있는 모든 웹, 앱, 데이터분석시스템들은
본질적으로는 모두가 mysql client들이다
