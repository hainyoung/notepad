# 프레임워크(FRAME WORK)란?
- 쉽게 말하면 프로그래밍을 쉽게 하기 위한 뼈대, 구조

EX)  파이썬의 웹 어플리케이션 제작 프레임워크 : 장고, 플라스크 등

* 참조 : https://m.blog.naver.com/manddonara/119738492

프레임워크 정의에서 내포하고 있는 의미 
: Application 개발에 바탕이 되는 템플릿과 같은 역할을 하는 클래스들과 인터페이스의 집합
즉, 프로그램의 전체적인 구조와 흐름을 확정지으며, 프로그래머가 정의해야 할 method의
이름까지도 결정짓는 역할을 한다

프레임워크란 공장과 같다
옷 공장, 신발 공장, 만두 공장 등 공장마다 서로 다른 제품을 생산한다
제품을 만들기 위해 어떤 재료를 사용하냐에 따라 같은 제품일지라도 품질, 디자인 등 여러 면이 달라진다
하지만, 공장이라는 개념을 봤을 경우, 공장은 재료를 준비하고 가공하고 생산하는 원리로 돌아간다

* 참조 : https://galid1.tistory.com/191

프레임워크란?
- 어떤 프로그램을 만들기 위한 기본 틀
- 사전적 정의 : 소프트웨어의 구체적인 부분에 해당하는 설계와 구현을 재사용이 가능하게끔
일련의 협업화된 형태로 클래스들을 제공하는 것

- 예시 : 컴퓨터 본체를 조립하는 회사라고 했을 때
1) 메인보드에 CPU를 장착하고
2) 메인메모리를 달고
3) 하드디스크를 장착하는 등의
이처럼 본체를 조립하기 위한 일련의 과정 또는 틀을 프레잉뭐크라고 한다

- 장점 
1) 프레임 워크를 사용하면 프로그램을 편리하고 빠르게 만들 수 있다(생산성 향상)
2) 체계적으로 만들어진다(유지보수가 쉽다)

라이브러리란?
- 재사용이 가능한 필요기능으로 반복적인 코드 작성을 없애기 위해 언제든지 필요한 곳에서
호출하여 사용할 수 있도록 Class나 Function으로 만들어진 것

- 예시 : 컴퓨터 본체를 조립한다고 했을 때 우리는 직접 CPU, Memory, 메인보드, HDD 등을
만들지 않고 이미 만들어져 있는 것을 가져다 사용한다. 컴퓨터 본체를 조립하기 위해 필요한
CPU, Memory, 메인보드 등이 각각의 모듈이라고 보면 된다

- 장점
1) 코드의 중복을 줄인다(가독성, 유지보수 향상)
2) 프로그래밍 시간을 줄일 수 있다(생산성 향상)

* 프레임워크 vs 라이브러리
프레임워크 : 내가 개발을 하기 위해 지켜야하는 틀
라이브러리 : 내가 개발을 하기 위해 필요로하는, 미리 구현 되어 있는 대상




# darknet 이란?

C언어로 작성된 물체 인식 오픈 소스 신경망 프레임워크이다
CUDA기반의 오픈 소스

주요 특징 
- 고속, 쉬운 설치, CPU 및 GPU 모두 지원
- YOLO (영상에서 실시간 물체 인식 가능)
- RNN

darknet 단일 이미지 인식 명령어
darknet detect cfg/yolov3.cfg yolov3.weights data/dog.jpg

darknet 비디오 인식 명령어
darknet detector demo cfg/coco.data cfg/yolov3.cfg yolov3.weights videofilepath.mp4

- 이미지의 경우 : detect
- 동영상의 경우 : detector 




# darkflow 란?

YOLO의 tensorflow 버전
기본 YOLO는 C언어를 기반으로 작성되었다
이를 Tensorflow로 작성한 것이 darkflow 이다

# YOLO(You only look once)

YOLO는 물체 탐지로 사용되는 알고리즘이다

how it works?
이전 탐지 시스템은 classifier나 localizer를 사용해 탐지를 수행
but YOLO는 하나의 신경망을 전체 이미지에 적용한다
이 신경망은 이미지를 영역으로 분할하고 각 영역의 bounding box와 확률을 예측한다
이런 bounding box는 예측된 확률에 의해 가중치가 적용된다

advantage
테스트를 할 때 이미지의 전체를 보고 예측정보를 알려준다
수천개의 이미지가 필요한 R-CNN과 달리 하나의 네트워크 평가로 예측할 수 있다
R-CNN보다는 1000배 이상 빠르고 Fast R-CNN보다는 100배 빠르다
