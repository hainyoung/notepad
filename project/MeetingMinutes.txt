# 3차 회의 
- 13:30 강의실에서 진행
- 구글링으로 각자 찾은 데이터 공유

- 1. Google에서 제공하는 Audioset 
- 유튜브 영상을 주제별로 라벨링하여 제공
- 데이터로 쓰기 위해 텐서플로 1.xx version 사용 -> 팀원의 시도 -> 데이터들 PCA 적용된 것으로 추측 
- 손실이 클 것으로 예상, 데이터로 쓰려면 '크롤링'하는 것이 더 나을 것이라는 판단

- 2. 공개데이터셋
- 스피치 데이터, 악기 소리 데이터 등이 있음(링크 들어가면 존재하지 않는 페이지도 꽤 있음)
- 어떻게 써 먹어야 할 지 감이 잘 안 잡히는 데이터들
- 영어 단어, 문장들의 음성 데이터는 사실상 쓸모가 없다고 판단(개인적으로)

- 3. Musicnet
- Imagenet 과 같이 음성데이터를 제공해주는 곳
- 악기 소리 데이터뿐
- 악기 소리 분류는 개인 프로젝트에선 쓸 만 했을 것 같은데 팀 프로젝트에선 조금 애매한 감이 있다고 판단
- 팀프로젝트 결과물에 '실용성'을 무조건 추구하는 건 아니지만(그렇게 결과물을 만들기도 사실상 무리가 있고)
- 그래도 어느 정도는 '음~ 고치면 다양하게 쓸 만하겠네' 라는 생각이 들게끔 해야 하는데
- 악기 분류는 팀 프로젝트 주제로는 과연 적합할까? 고민을 하게 만드는 아이템

- 아이디어 제안
- 1. 이미지에 따른 음악 추천
- 내가 조심스레 제안 해 본 주제(이미지 분류가 main이 되는 느낌이라 조심스러웠다)
- 음성 데이터 구글링 하다가 음성 인식 기술로 어떤 신박한 게 만들어졌는지에 관한 기사를 보고 착안
- 기사에서는 크리스마스 트리 이미지를 입력하면 크리스마스에 어울리는 노래를 작곡해주는 인공지능 기술을 소개
- 노래 작곡은 말도 안 되는 것이고
- 우리가 할 수 있는 게 무엇일까,,, 생각하다가 떠올렸다
- 생일 케이크를 찍은 사진을 입력하면 생일에 관련한 노래를 추천 또는 재생하는 프로그램을 만들어 볼 수 있지 않을까?
- 노을 사진, 비 오는 사진, 카페 사진 또는 커피 등의 사진을 내가 찍었을 때 그에 맞는 음악이 추천 또는 재생 되게!
- 이렇게 하면 이미지 분류가 되어 버리는 것 같아서 음성 쪽을 융합하려면 더 살을 붙여야 한다는 것에 모두 동의
- 현재 100% 음성 인식을 주제로 하기에는 조금 길이 보이지 않는 느낌이 있어서 이런 아이디어를 후보상에 두면 좋겠다는 것에 동의
- 추가 아이디어 : 이미지뿐만 아니라 동영상 쪽도 건드려 볼 수 있을 것 같다

- 2. 우리가 말한 문장을 시 형식으로 출력 또는 랩Rap으로 출력
- 쉽게 예를 들어, "나는 오늘 김밥을 먹었다" 라고 말하면 (또는 문자로 입력하면) 운율을 갖춘, 느낌 있는 '시' 로 만들어주는 것
- 아니면 "나는 오늘 김밥을 먹었다" 라고 말하면 비트와 함께 라임을 갖춘 랩으로 만들어주는 것
- 아이디어 진짜 대박!
- 하지만 실현하기가 많이 힘들어 보인다는 의견
- 만약 '말'로 한 것을 저렇게 출력하려면 먼저 TTS 기술을 써야 함
- API가 있다고 하는데 팀 프로젝트에서 사용을 해도 되는 것인지 정확히 판단이 잘 안 됨
- TTS 기술 없이 우리가 직접 기계에 입력한다고 하면, 자연어처리를 해야 함
- 구현기술에 관하여 명확한 것이 좀 부족함'

- 결론 : 우리의 기술적인 한계를 늘 마주하게 되는 아이디어 회의
